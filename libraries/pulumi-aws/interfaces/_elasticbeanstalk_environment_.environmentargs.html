Optional tier

tier: pulumi.ComputedValue<string>
  • Defined in elasticbeanstalk/environment.ts:191

Elastic Beanstalk Environment tier. Valid values are Worker or WebServer. If tier is left blank WebServer will be used.

  • setting – (Optional) Option settings to configure the new Environment. These override specific values that are set as defaults. The format is detailed below in Option Settings
  • solution_stack_name – (Optional) A solution stack to base your environment off of. Example stacks can be found in the [Amazon API documentation][1]

Optional waitForReadyTimeout

waitForReadyTimeout: pulumi.ComputedValue<string>
  • Defined in elasticbeanstalk/environment.ts:210

The maximum duration that Terraform should wait for an Elastic Beanstalk Environment to be in a ready state before timing out.

  • poll_interval – The time between polling the AWS API to check if changes have been applied. Use this to adjust the rate of API calls for any create or update action. Minimum 10s, maximum 180s. Omit this to use the default behavior, which is an exponential backoff
