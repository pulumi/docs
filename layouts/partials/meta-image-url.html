{{- /* Returns the effective social/meta image for a page as a site-relative URL, or "" if none applies. Single source of truth shared by head.html (OpenGraph / Twitter tags) and the schema collectors (JSON-LD image), so both stay in sync. Precedence: 1. A page-level `meta_image` (custom override — always wins). 2. A committed, build-time generated card at assets/images/generated/.png, if one exists. The path mirrors the page's content path (.md and trailing /_index stripped), matching scripts/generate-meta-images.mjs — so it works for flat sections (what-is) and nested ones (docs). 3. "" (caller falls back to its own default). Generated cards are produced by scripts/generate-meta-images.mjs. The lookup is automatic: enabling a section is just a matter of generating + committing its cards — no template change needed. */ -}} {{- $img := "" -}} {{- if .Params.meta_image -}} {{- if or (eq .Section "blog") (eq .Section "templates") -}} {{- with .Resources.GetMatch .Params.meta_image -}}{{- $img = .RelPermalink -}}{{- end -}} {{- else -}} {{- $img = .Params.meta_image -}} {{- end -}} {{- else -}} {{- with .File -}} {{- $key := .Path | replaceRE "\\.md$" "" | replaceRE "/_index$" "" -}} {{- with resources.Get (printf "images/generated/%s.png" $key) -}} {{- $img = .RelPermalink -}} {{- end -}} {{- end -}} {{- end -}} {{- return $img -}}
