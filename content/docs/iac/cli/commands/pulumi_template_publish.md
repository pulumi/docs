Publish a template to the Private Registry

## Synopsis

Publish a template to the Private Registry.

This command publishes a template directory to your organization's Private Registry. Registry-backed templates provide semantic versioning, immutable storage, and reliable access for your organization's template library.

The template directory must contain a valid `Pulumi.yaml` file with a `template` section. Optionally include a `README.md` for documentation that will be displayed in the Private Registry.

```
pulumi template publish <directory> [flags]
```

## Examples

```bash
# Publish a template with version 1.0.0
pulumi template publish ./my-template --name my-template --version 1.0.0

# Publish to a specific organization
pulumi template publish ./my-template --name my-template --version 1.2.3 --publisher my-org

# Publish a new version of an existing template
pulumi template publish ./my-template --name my-template --version 2.0.0
```

## Options

```
  -h, --help               help for publish
      --name string        The name of the template (required)
      --publisher string   The publisher of the template (e.g., 'pulumi'). Defaults to the default organization in your pulumi config.
      --version string     The version of the template (required, semver format)
```

## Requirements

- Template directory: Must contain a valid `Pulumi.yaml` file at the root
- Template section: The `Pulumi.yaml` must include a `template` section
- Version format: Must follow semantic versioning (e.g., "1.0.0", "2.1.3")

## Template structure

A valid template directory structure:

```
my-template/
├── Pulumi.yaml          # Required: Must include template section
├── README.md            # Optional: Documentation for the template
├── index.ts             # Template source files
├── package.json
└── ...
```

The `Pulumi.yaml` must include a `template` section:

```yaml
name: my-project
runtime: nodejs
description: My project description
template:
  displayName: "My Template"
  description: "A template for..."
  config:
    region:
      description: "Deployment region"
      default: "us-west-2"
```

## Using published templates

Once published, templates can be used with `pulumi new`:

```bash
# Use latest version
pulumi new private/my-org/my-template

# Use specific version
pulumi new private/my-org/my-template@1.0.0
```

Templates are also available in the Pulumi Console's Private Registry and New Project Wizard.
